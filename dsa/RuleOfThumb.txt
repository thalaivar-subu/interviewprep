THE CORE MOVE
  Read every question as:   OBJECTIVE  +  STRUCTURE  +  CONSTRAINT
    OBJECTIVE  (min / max / longest / count / all / exists / kth)
               = a DECOY. It says what to RETURN, never the pattern.
    STRUCTURE  (array / string / tree / graph / grid / intervals / list / words)
    CONSTRAINT (contiguous? can-skip? sorted? weighted? reuse? adjacent?)
  THE PATTERN = STRUCTURE + CONSTRAINT.  Decide those, THEN think in the pattern.

------------------------------------------------------------------------
DECISION TREE  (text version of pattern-flow.svg)
------------------------------------------------------------------------
STEP 1 — what STRUCTURE is the input?
  TREE .......... per level ................. BFS (queue)
                 path / subtree property .... DFS (recursion)
  GRAPH ......... weighted edges ............ Dijkstra
                 depends-on / order ......... Topological Sort
                 groups / connected? ........ Union-Find
                 1 step / grid / islands .... BFS or DFS
  LINKED LIST ... cycle / middle ............ Fast & Slow pointers
                 reorder / remove ........... Reverse in place (+dummy head)
  INTERVALS ..... [start, end] pairs ........ Intervals (sort + sweep)
  WORDS ......... prefix / autocomplete ..... Trie
  ARRAY/STRING .. ........................... go to STEP 2

STEP 2 — array / string: what does it ASK?
  sorted, or "smallest x where cond flips" .. Binary Search
  contiguous window · SUM+/- or COUNT ....... Prefix Sum + HashMap
  contiguous window · FIXED size k .......... Sliding Window (fixed)
  contiguous window · LONGEST / SHORTEST .... Sliding Window (dynamic)
  next greater / smaller .................... Monotonic Stack
  max/min of EVERY window ................... Monotonic Deque
  seen? / how many? ......................... Hashing
  SUBSEQUENCE (may skip) .................... DP
  return ALL / list every ................... Backtracking
  optimize AND subproblems repeat ........... DP
  one safe local choice, never undo ......... Greedy

WATCH-OUTS — the splits that trip people up
------------------------------------------------------------------------
1) Goal word is a decoy. "longest/min/max/count" never picks the pattern.
2) Subarray SUM: negatives or "count" -> Prefix Sum + Hash ;
   all-positive & min/max length -> Sliding Window.
3) "subsequence" (skip ok) -> DP ; "substring/subarray" (contiguous) -> Window.
4) DP vs Greedy: same subproblem repeats when you recurse -> DP ;
   one local choice always safe -> Greedy.
   (unsure? try greedy, hunt a counterexample; if one exists -> DP.)
5) Partition / coins: "return all" -> Backtracking ; "count ways" -> DP ;
   "fewest / min" -> DP-min.
6) Tree PATH sum (LC437) = DFS traversal + Prefix-Sum hashmap TOGETHER,
   not one or the other.
7) Two Sum: unsorted -> Hashing ; sorted -> Two Pointers ;
   triplets summing to 0 -> sort + Two Pointers.
8) Count ways, order MATTERS -> DP loop target-outer [LC377] ;
   order DOESN'T -> DP loop coins-outer [LC518].
